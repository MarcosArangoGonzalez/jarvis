## Modo Solo Activo

### Optimizaciones
- Respuestas cortas y directas — sin scaffolding innecesario
- Commits más informales aceptados: `wip: draft session wizard`
- PR/review overhead reducido
- Menos documentación inline — más contexto en vault

### Velocidad sobre formalidad
- No crear ADR para decisiones pequeñas
- Tests cuando añadan valor real, no por cobertura métrica
- Refactor inline si es rápido y obvio

### Pero siempre
- Security scanner antes de commit
- Tests para lógica de negocio crítica
- Actualización del vault al cerrar sesión significativa
